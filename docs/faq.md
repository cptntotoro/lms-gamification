# FAQ — Часто задаваемые вопросы

**Геймификация образовательного процесса в LMS**

**Версия:** 0.1 (февраль 2026)  
**Дата обновления:** 23 февраля 2026  
**Проект:** lms-gamification  
**GitHub:** https://github.com/cptntotoro/lms-gamification

## Общие вопросы

**1. Что такое эта система геймификации?**  
Это модуль, который начисляет студентам очки и уровни за выполнение учебных активностей в LMS (тесты, задания, вебинары
и т.д.). Студенты видят прогресс в виджете, преподаватели настраивают правила, администраторы контролируют систему.

**2. Зачем нужна геймификация?**  
Повышает мотивацию студентов, увеличивает вовлечённость в задания, даёт преподавателям инструмент для дополнительной
стимуляции.

**3. Какие LMS поддерживаются?**  
Любые LMS, которые могут отправлять POST-запросы на webhook (`/api/events`). Тестировалось с Moodle, Open edX, 1С:
Университет.

## Для студентов

**4. Где я могу увидеть свои очки и уровень?**  
В личном кабинете LMS или на странице курса — обычно в виде виджета (уровень, очки, прогресс до следующего уровня).

**5. Почему очки не начисляются?**  
Возможные причины:

- Тип активности отключён администратором
- Превышен дневной лимит по этому типу
- Событие уже было обработано ранее (дубликат)
- Ошибка в настройках LMS (неправильный `typeCode`)

**6. Можно ли посмотреть полную историю начислений?**  
В текущей версии виджет показывает только последние 3–5 начислений. Полная история доступна администратору LMS.

**7. Можно ли потерять очки?**  
Пока нет. Снятие/штрафы очков запланированы на весну 2026.

## Для преподавателей

**8. Как настроить начисление очков за свои задания?**  
Зайдите в админ-панель LMS → Геймификация → Типы событий.  
Создайте новый тип:

- Укажите уникальный `typeCode` (например `homework-math-1`)
- Название для студентов
- Количество очков
- Дневной лимит (если нужен)
- Включите «Активно»

**9. Можно ли задать разные очки для одного типа активности на разных курсах?**  
Пока нет — очки задаются глобально для типа события. Поддержка правил по курсу/группе планируется на 2026 год.

**10. Как временно отключить начисление за определённый тип?**  
В разделе Типы событий найдите нужный тип → снимите галочку «Активно».  
Начисления прекратятся, история сохранится.

## Для администраторов LMS

**11. Как подключить геймификацию к LMS?**  
Настройте webhook в LMS:

- URL: `https://ваш-сервер/api/events`
- Метод: POST
- Тело запроса: JSON с `userId`, `eventId`, `eventType`

**12. Что делать, если события приходят, но очки не начисляются?**

- Проверьте логи LMS (отправляется ли запрос)
- Проверьте ответ сервера (200 OK + JSON)
- В логах сервера найдите `eventId` и причину (дубликат, лимит, отключён тип)

**13. Почему иногда приходит ответ "duplicate"?**  
LMS отправляет одно событие несколько раз.  
Исправьте логику в LMS — событие должно отправляться ровно один раз.

**14. Как временно отключить всю геймификацию?**  
Отключите все типы событий (`active = false`) в админке → Типы событий.

**15. Можно ли удалить тип события?**  
Физическое удаление не предусмотрено — только деактивация (`active = false`). Это сохраняет историю начислений.

**16. Где посмотреть все начисления конкретного студента?**  
GET `/api/admin/users/{userId}/transactions` — полная история с пагинацией и фильтрами.

**17. Что делать при ошибке "500 Internal Server Error"?**  
Проверьте логи сервера геймификации.  
Создайте issue на GitHub с описанием и логами: https://github.com/cptntotoro/lms-gamification/issues

## Технические вопросы

**18. Какие версии технологий используются?**

- Java 21
- Spring Boot 3.2+
- PostgreSQL 17+
- Hibernate + Spring Data JPA

**19. Как обновить систему до новой версии?**

- Сделайте бэкап БД
- Обновите код (`git pull`)
- Выполните `mvn clean package`
- Перезапустите приложение
